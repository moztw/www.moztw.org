Thunderbird 版本資訊

2019 年 8 月 27 日 v.68.0

此版本資訊包含了 Thunderbird 新鮮事、已知問題、下載安裝指示、以及常見問題等訊息。

我們想知道您對 Thunderbird 的想法。歡迎到 MozillaZine 上的英文討論區或 MozTW 的 Thunderbird 中文討論區與我們分享意見。 若您有興趣的話，也可至 Bugzilla 看看這個版本的完整變更清單。

Thunderbird 新鮮事

  • 新增 現在起可以重新連結雲端鏈結附件，不需重新上傳。
  • 新增 可將單一帳號的所有信件匣標示為已讀。
  • 新增 可定期執行過濾器，也改善了過濾器執行紀錄。
  • 新增 針對 Yandex 支援 OAuth2 驗證。
  • 新增 現在起，可在「進階選項」中選擇語言套件。（需要先設定 intl.multilingual.enabled 偏好設定，可能還需要將 extensions.langpacks.signatures.required 設定為 false）
  • 新增 針對 Windows 平台推出 64 位元一般安裝程式及 MSI 安裝程式。請參考 Thunderbird for Organisations 頁面當中的詳細資訊。
  • 新增 加入可於企業環境使用 Windows 群組原則或跨平台 JSON 檔案來自訂 Thunderbird 部署的政策引擎。
  • 新增 針對 IMAP 通訊協定支援 TCP keepalive。
  • 新增 針對 MAPI 介面完整支援 Unicode：新增支援 MAPISendMailW。
  • 新增 為了保護您的設定檔資料不受舊版程式破壞，Thunderbird 68 版起，會顯示「您啟動了舊版的 Thunderbird」的訊息，且不允許開啟某些設定檔。暫時替代方法：加入 --allow-downgrade 參數啟動。若需詳細資訊，請參考這篇技術支援文章。
  • 新增 行事曆：現在起，時區資料會包含過去與未來的異動資料。已包含所有於 2018-2022 年間的異動資料。
  • 新增 聊天：現在起，可於每個對話中指定不同的拼字檢查語言。
  • 變更 附加元件支援：僅支援作者修改過的附加元件。
  • 變更 字典支援：現在起僅支援 WebExtension 格式的字典。現在起 addons.mozilla.org 及 addons.thunderbird.net 皆提供 WebExtension 格式的字典。
  • 變更 佈景主題支援：現在起僅支援 WebExtension 格式的佈景主題。現在起 addons.mozilla.org 及 addons.thunderbird.net 皆提供 WebExtension 格式的佈景主題。
  • 變更 已移除獨立的選項視窗，現在起所有選項都會顯示於分頁中。
  • 變更 調整了安裝附加元件時的介面。
  • 變更 針對佈景主題有許多改進，包含暗色訊息清單、討論串窗格。
  • 變更 應用程式選單，或稱「漢堡選單」。
  • 變更 改善寫信視窗當中的輸入、選擇及移除收件者的功能。
  • 變更 檔案鏈結功能：加入 WeTransfer。另可安裝使用新版 WebExtensions FileLink API 的 Dropbox 及 Box.com 的附加元件。
  • 變更 寫信視窗的文字顏色、背景色、標籤自訂色彩不再受限於 10x7 種 HTML 色彩。
  • 變更 預設不再送出編輯文字色彩與背景色。於「工具 -> 選項、編輯」加入新選項。
  • 變更 針對外部與取消附加的附件介面做改進。現在起以鏈結顯示，且可使用新選單選項「開啟所在資料夾」。
  • 變更 不再於行內顯示文字附件（純文字、HTML、XML 等格式）。可切換 mail.inline_attachments.text 偏好設定即可再次於行內顯示。
  • 變更 改進某些格式的釣魚信件偵測功能。
  • 變更 改進詐騙郵件警告功能。
  • 變更 回信給自己時會搜尋所有身分（mailnews.reply_to_self_check_all_ident 偏好設定預設值更改為 true）
  • 變更 Maildir 現在起改用 message ID 作為主檔名及 .eml 副檔名
  • 變更 自動壓實重整功能門檻，從 20 MB 調整為 200 MB。
  • 變更 行事曆：改善事件對話框的使用者介面。
  • 變更 行事曆：Thunderbird 的行事曆附加元件 Lightning 現在起使用與 Thunderbird 相同的版本編號方式。
  • 修正 重新啟動後，會遺失「搜尋訊息」對話框中的欄位選擇與排序順序。
  • 修正 某些情況下，全域索引器會造成高 CPU 資源占用。
  • 修正 有時候其他使用者看不到設定在 IMAP 信件匣的標籤。
  • 修正 附加元件詳細資訊的「贊助」按鈕無法正常運作。
  • 修正 帳號刪除或伺服器或帳號名稱變更後，不會清除 SMTP 密碼。
  • 修正 若開啟 Windows 搜尋整合功能後，無法從 Mbox 轉換為 Maildir。
  • 修正 使用 Maildir 的情況下，不會從垃圾桶清除郵件。
    註：其他與 Maildir 相關的刪除問題可能仍然存在。
  • 修正 因安全性問題造成訊息傳送失敗時，僅會顯示「未知錯誤」。現在會提供更多資訊。
  • 修正 改進 SMTP 5.7.1 訊息。
  • 修正 行事曆：無法以多個使用者帳號對同一 CalDAV 伺服器進行連線。
  • 修正 聊天：修正使用 SASL 的 XMPP 驗證問題。

已知問題

  • 尚未修復 編輯標籤功能無法正常運作（將於 68.1 版修正）
  • 尚未修復 無法正常於「搜尋訊息」結果對話框移動訊息（將於 68.1 版修正）
  • 尚未修復 收到的附件清單當中的內容類型/操作的相關使用問題（將於 68.1 版修正）

下載與安裝

系統需求

在安裝 Thunderbird 前請先確定您的電腦是否合乎系統需求。

下載 Thunderbird

您可以由此下載最新的 Thunderbird，或取得 Windows、Linux 及 Mac OS X 平台上的各國語言版本，或由此下載 Thunderbird 未來發行版本（測試版）。若需自行編譯其他平台或語言的版本，請見本文件最末節。

安裝 Thunderbird

請注意：安裝新版 Thunderbird 將 覆蓋現存的 Thunderbird 程式，您不會遺失書籤或瀏覽記錄，但有些附加元件可能要更新才能繼續使用。 詳細安裝步驟請參見如何安裝 Thunderbird。

移除 Thunderbird

在 Windows 作業系統中，您可以使用「新增或移除程式」移除 Thunderbird。OS X 的使用者請將 Thunderbird 程式移至垃圾筒；Linux 使用者則請移除包含 Thunderbird 的資料夾。

一般狀況下，移除 Thunderbird 時不會把書籤、瀏覽記錄、擴充套件和其他附加元件一併刪除， 這些資料會保留在您的「個人設定資料夾」內， 此資料夾可以在「說明」選單的「疑難排解資訊…」找到， 按下「應用程式一般資訊」中「設定檔目錄」旁的按鈕便可開啟它。

請留意若您沒有手動刪除個人設定資料夾，移除後重新安裝 Thunderbird 將繼續沿用原本的設定檔。

附加元件及佈景主題

您可以由「附加元件管理員」或附加元件網站取得附加元件及佈景主題。
