Thunderbird 2 是頗受好評的 Mozilla Thunderbird 電子郵件軟體第二版。

此版本資訊包含 Thunderbird 2 的新鮮事、下載與安裝指示、已知問題以及技術支援與意見回饋等訊息。如發現任何程式錯誤，請先行將版本資訊與錯誤回報注意事項閱讀完畢，再到 Bugzilla 回報。

我們想知道您對 Thunderbird 的意見。您可填寫回報表單，或到討論區討論相關議題：您可至 MozillaZine 上的英文討論區或 MozillaTaiwan 的 Thunderbird 中文討論區與我們分享意見。

系統需求

在安裝 Thunderbird 前請先確定您的電腦合乎系統需求。 從 1.5 開始，Mac OS X 用戶必須在 MacOX X 10.2 以上版本運行本程式。

下載 Thunderbird

Mozilla.org 提供 Windows 版、Linux 版和 Mac OS X 版多種語言的 Thunderbird。您可由此下載最新的 Thunderbird 2。若想自行編譯其他不支援的平台或語言，請見本文件最末節。

安裝 Thunderbird

請注意：安裝 Thunderbird 2 將覆蓋您現存的 Thunderbird 程式，這並不會遺失你的任何郵件，但有些附加元件可能需要更新才能繼續使用。

移除 Thunderbird

在 Windows 作業系統中，您可以自「開始」功能表中的「控制台」移除 Thunderbird 2。OS X 的使用者請直接移除「Thunderbird」應用程式，Linux 使用者則請移除「thunderbird」資料夾。

移除 Thunderbird 2 並不會將你的任何郵件和其他附加元件一併刪除，這些資料會留在您的「個人設定資料夾」中，此資料夾存放位置則視作業系統而定：

Windows Vista Users\<使用者名稱>\AppData\Roaming\Thunderbird
Windows 2000, XP Documents and Settings\<使用者名稱>\Application Data\Thunderbird
Windows NT WINNT\Profiles\<使用者名稱>\Application Data\Thunderbird
Windows 98, ME Windows\Application Data\Thunderbird
Mac OS X ~/Library/Thunderbird
Linux 及 Unix 系統 ~/.thunderbird

移除 Thunderbird 2 後所安裝的任一版本 Thunderbird 將繼續使用「個人設定資料夾」中的資料。

擴充套件及佈景主題

Thunderbird 1.5 安裝的擴充套件可能已不相容於 Thunderbird 2 而應加以升級，如發生此類問題，請回報附加元件原作者。一旦安裝 Thunderbird 2，所有擴充套件及佈景主題會先停用，待 Thunderbird 2 偵測出此套件無相容問題或能升級才會啟用。

已知問題

以下列出 Thunderbird 2 部分已知問題，請閱讀後再回報新的程式錯誤，並請定期檢查本欄的更新內容 （如在此版本發現新問題，我們會隨時在英文網頁更新此欄）。

所有系統
  • 某些軟體防火牆執行時會悄悄封鎖 Thunderbird 的對外連線，此類問題通常會在 Thunderbird 升級或自舊版覆蓋安裝時發生。您可參考防火牆設定步驟確保 Thunderbird 於網路上通行無阻。
  • 預設的拼字檢查字典只有英文，其它非英文的字典可以用擴充套件的形式安裝，請至 addons.mozilla.org 下載。
  • 「郵件檢視」工具列按鈕不再是預設按鈕，欲使用此一功能，你需要手動將它放回工具列：「檢視＞工具列＞自訂」。
  • 為了讓 Thunderbird 能依照你的網路連線自動調整連線狀態，你需要將 network.autodial-helper.enabled 設為「false」。
  • 在全域收件匣和 POP3 帳號的收件匣之間切換時，請確定已將郵件備份到其他資料夾。
  • 提供給擴充套件和佈景主題開發者使用的新 DOM 檢視器可用擴充套件的形式安裝：https://addons.mozilla.org/thunderbird/1806/。
Mac OS X
  • 在 Intel 核心的麥金塔電腦上執行時，不論原生執行或是透過 Rosetta 執行，都沒有 Talkback（錯誤回報）功能。當錯誤發生時，Apple Crash report 程式會自動執行。
Linux 和 Unix 系統
  • 若 Thunderbird 的安裝路徑中含有空白字元，可能便無法設定為預設瀏覽器、並將於啟動時不斷詢問是否設定。目前的解決方法是安裝到無空白字元的路徑下。
  • 在 Fedora Core 3 無法與 GNOME 正常整合。 Fedora Core 3 的使用者必須先下載 linc-1.0.3-3.1.i386.rpm。 下載後安裝，安裝完畢後在 Thunderbird 的安裝目錄下執行此命令 (需要寫入權限)：

    touch .autoreg

    下回使用 Thunderbird 時，便應該能正確整合至 GNOME 環境。

疑難排解

  • 設計不良或不相容的套件將導致瀏覽器發生問題，發生的問題包含當機或網頁瀏覽速度變慢等等。如果您碰上收信軟體無法開啟、視窗顯示不正常等等狀況，便可能是擴充套件或佈景主題的問題。這時，請將收信軟體重新啟動在安全模式：
    • 在 Windows 下用「Safe Mode」捷徑或在命令列中輸入 thunderbird.exe -safe-mode 指令。
    • 在 Linux 下請執行 ./thunderbird -safe-mode 指令。
    • 在 MacOS X 下請輸入

      cd /Applications/Thunderbird.app/Contents/MacOS/
      ./thunderbird-bin -safe-mode

      當啟動安全模式時，所有的附加元件將停用並使用預設的佈景主題。在安全模式下移除有問題的擴充套件/佈景主題，然後再正常的啟動瀏覽器。

常見問題集

  1. 我該怎麼幫忙呢？

    我們需要大量曝光。讓推廣 Thunderbird 成為您的任務，使您的親朋好友上司下屬都認識它： 如果您是個學生，就讓它廣佈校園之中；寫篇文章投稿至 Slashdot 等新聞站台上，發布 Thunderbird 新版的新聞；也可以在您的 Blog 上透漏一點相關訊息；總之，儘可能的在網際網路中散佈就對了！

  2. 你們為什麼都沒有回信？

    請使用討論區， Thunderbird 團隊會定期閱讀上面的文章。我們常常都收到大量的信件，而您的可能不小心漏失了。

  3. 我該去哪裡取得新的佈景主題與擴充功能（附加元件）？

    擴充套件及佈景主題可自 Thunderbird 附加元件下載。

  4. 哪些人正致力於 Thunderbird 的開發工作？

    很多人。詳細列表請參考「說明＞關於 Mozilla Thunderbird」

  5. Thunderbird 2 的原始碼在何處？

    Thunderbird 2 原始碼已開放下載，而目前開發中的最新程式碼則需以 CVS 取得。Thunderbird 相關程式為「mozilla/mail」、「mozilla/mailnews」、「mozilla/toolkit」及「mozilla/chrome」，請參考建置步驟。

非官方建製版

Mozilla 代表各國作者提供了各國語系的地方化版本，您可至Thunderbird 2 下載頁取得檔案。

其他尚未被官方 Firefox 2 驗證通過的本地化版本可以在 FTP 站台取得。
