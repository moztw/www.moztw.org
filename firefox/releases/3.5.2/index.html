Firefox 3.5.2 新鮮事

Firefox 3.5.2 修正了以下問題：

  • 許多安全性問題。
  • 具 ICC 設定檔的圖像可以在所有顯示器上正確顯示了。

請參考此版本的完整變更清單。或前往 Firefox 3.5.1 新鮮事瞭解前一個版本的更新。

欲知 Firefox 3.5 的所有功能，請見 Firefox 網頁。

下載與安裝

系統需求

在安裝 Firefox 3.5 前請先確定您的電腦是否合乎 系統需求。

下載 Firefox 3.5

Mozilla.org 提供了 Windows、Linux 及 Mac OS X 平台上多種語言版的 Firefox 3.5，您可由此下載最新的 Firefox 3.5 。若想自行編譯其他不支援的平台或語言，請見本文件最末節。

安裝 Firefox 3

請注意：安裝 Firefox 3.5 將覆蓋您現存的 Firefox 程式，這並不會造成書籤或瀏覽記錄遺失，但可能會有些附加元件要更新才能繼續使用。您可以稍後重新安裝舊版本，如果想要降級的話。

移除 Firefox 3.5

在 Windows 作業系統中，您可以自「開始」選單中的「控制台」移除 Firefox 3.5。OS X 的使用者請移除「Firefox」 程式、Linux 使用者則請移除「firefox」 資料夾。

移除 Firefox 3.5 並不會將書籤、瀏覽記錄、擴充套件和其他附加元件一併刪除，這些資料會留在您的「個人設定資料夾」中，此資料夾存放位置則視作業系統而定：

Windows Vista Users\<UserName>\AppData\Roaming\Mozilla\Firefox
Windows 2000, XP, Server 2003 Documents and Settings\<UserName>\Application Data\Mozilla\Firefox
Mac OS X ~/Library/Application Support/Firefox
Linux and Unix systems ~/.mozilla/firefox

移除 Firefox 3.5 後所安裝的任一版本 Firefox 將繼續使用個人設定資料夾中的資料。

擴充套件及佈景主題

Firefox 3.5 安裝的擴充套件可能已不相容於先前的版本而應加以升級，如發生此類問題，請回報附加元件原作者。一旦安裝 Firefox 3.5 ，所有擴充套件及佈景主題會先停用，待 Firefox 3.5 偵測出此套件無相容問題或能升級才會啟用。

已知問題

以下列出部分 Firefox 3.5 已知問題，請閱讀後再回報新的程式錯誤。

所有系統
  • 安裝此版本後，使用者將無法回復至舊的 Firefox 3.5 測試版本，除非建立新的使用者設定（見 bug 488966）。
  • 由備份中復原您的書籤可能會花點時間，其間瀏覽器可能會看似沒有回應（見 bug 493731）。
  • 使用「清除最近的歷史記錄」後，某些 SSL 網站將無法讀取所有圖片及樣式，除非重新載入（見 bug 480619）。
  • Firefox 3.5 將只支援 ICC v2 標準的色彩設定檔，後續版本將增加支援 ICC v4（見 bug 463221）。
  • 使用動態下載字型的網站，若要以 :hover 改變字體顏色，需新增 text-rendering : optimizeLegibility 以適用於字體大小 20 像素以下者（見 bug 495455）。
  • 有些使用 Flash 的網站可能引起 Cookie 對話框的問題（見 bug 495035）。
  • 網頁不再能自動安裝 PKCS1 加密符記。使用者現在必須手動，或安裝附加元件才能安裝（見 instructions and documentation）。
Windows 及 Windows Vista
  • 若您正在執行 AVG SafeSearch v8.0 或以下版本，在網址列按 Enter 將沒有任何動作。升級 AVG SafeSearch 可解決此問題（見 bug 479095）。
  • 若由列印對話框中選擇傳真，Windows 傳真對話框會出現然後消失。其他應用程式仍能正常進行傳真（見 bug 440486）。
Mac OS X
  • 若使用者重複變更分頁尺寸並要求預覽列印，Firefox 可能會當機（見 bug 495567）。
Linux 及 Unix
  • 在某些網站縮小檢視尺寸可能會出現灰或黑線（bug 477552）。

疑難排解

  • 設計不良或不相容的套件將導致瀏覽器發生問題，發生的問題包含當機或網頁瀏覽速度變慢等等。如果您碰上瀏覽器無法開啟、視窗顯示不正常等等狀況，便可能是擴充套件或佈景主題的問題。這時，請將瀏覽器重新啟動在安全模式：
  • 在 Windows 下用「Safe Mode」捷徑或在命令列中輸入 firefox.exe -safe-mode 指令。
  • 在 Linux 下請執行./firefox -safe-mode 指令。
  • 在 MacOS X 下請輸入

    cd /Applications/Firefox.app/Contents/MacOS/
    ./firefox-bin -safe-mode

    當啟動安全模式時，所有的附加元件將停用並使用預設的佈景主題。在安全模式下移除有問題的擴充套件/佈景主題，然後再正常的啟動瀏覽器。
  • 如果您想在移除某擴充套件後，以 -install-global-extension 指令將同一擴充套件裝到每個使用者設定檔中，則必須在重新安裝前先重開瀏覽器一次，以便徹底移除擴充套件。如果您沒這麼做，則擴充套件清單可能會有衝突項目而不能為所有設定檔安裝。
  • 如果您遇到的怪問題與書籤、下載、視窗位置、工具列、瀏覽記錄或其他設定項目有關，建議您在回報問題前先行建立新的使用者設定檔，並查看能否重現該問題。建立新設定檔的方式如下：執行 Firefox 時加上 -P 的命令列參數，選擇「管理使用者設定檔」按鈕，然後選擇「建立使用者設定檔」。將各種設定(書籤、儲存的密碼等)從舊的設定檔中分次轉移到新的設定檔，每移轉一項便檢查是否發生問題。如果您發現某特定設定檔資料會造成問題，請回報錯誤並且附上該檔案。

常見問題集

  1. 我該怎麼幫忙呢？

    我們需要開發員和測試社群回饋建議，幫我們將 Firefox 變得更盡善盡美。請先閱讀錯誤回報注意事項，再到 Bugzilla 回報。您也可以使用回饋表單表達意見。

  2. 你們為什麼都沒有回信？

    請使用討論區， Firefox 團隊會定期閱讀上面的文章。我們常常都收到大量的信件，而您的可能不小心漏失了。

  3. 我該去哪裡取得新的佈景主題與擴充功能 (附加元件)？

    擴充套件及佈景主題可自 Mozilla 附加元件中心下載。

  4. Firefox 3.5 是哪些人做出來的？

    很多人。從「說明＞關於 Mozilla Firefox」中有份名單，列出部分曾對 Firefox 3.5 有所貢獻的人。

  5. Firefox 3.5 的原始碼在何處？

    Firefox 3.5 原始碼已開放下載，而目前開發中的最新程式碼則需以 CVS 取得。Firefox 相關程式為「mozilla/browser」、「mozilla/toolkit」及「mozilla/chrome」，請參考建製步驟。

  6. 電子郵件程式呢？

    Firefox 3.5 能與您預設的任何郵件程式協同運作， 而我們推薦您試試我們下一代的郵件軟體：與 Firefox 合作無間的 Mozilla Thunderbird。

非官方建製版

Mozilla 代表各國作者提供了各國語系的地方化版本，您可至Firefox 3.5 下載頁取得檔案。

其他尚未被官方 Firefox 3.5 驗證通過的本地化版本可以在官方站台取得。
