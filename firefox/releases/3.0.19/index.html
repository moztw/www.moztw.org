下載與安裝

系統需求

在安裝 Firefox 3 前請先確定您的電腦是否合乎 系統需求。

下載 Firefox 3

Mozilla.org 提供了 Windows、Linux 及 Mac OS X 平台上多種語言版的 Firefox 3，您可由此下載最新的 Firefox 3 。若想自行編譯其他不支援的平台或語言，請見本文件最末節。

安裝 Firefox 3

請注意：安裝 Firefox 3 將覆蓋您現存的 Firefox 程式，這並不會造成書籤或瀏覽記錄遺失，但可能會有些附加元件要更新才能繼續使用。

移除 Firefox 3

在 Windows 作業系統中，您可以自「開始」選單中的「控制台」移除 Firefox 3。OS X 的使用者請移除「Firefox」 程式、Linux 使用者則請移除「firefox」 資料夾。

移除 Firefox 3 並不會將書籤、瀏覽記錄、擴充套件和其他附加元件一併刪除，這些資料會留在您的「個人設定資料夾」中，此資料夾存放位置則視作業系統而定：

Windows Vista Users\<UserName>\AppData\Roaming\Mozilla\Firefox
Windows 2000, XP, Server 2003 Documents and Settings\<UserName>\Application Data\Mozilla\Firefox
Mac OS X ~/Library/Application Support/Firefox
Linux and Unix systems ~/.mozilla/firefox

移除 Firefox 3 後所安裝的任一版本 Firefox 將繼續使用個人設定資料夾中的資料。

擴充套件及佈景主題

Firefox 3 安裝的擴充套件可能已不相容於先前的版本而應加以升級，如發生此類問題，請回報附加元件原作者。一旦安裝 Firefox 3 ，所有擴充套件及佈景主題會先停用，待 Firefox 3 偵測出此套件無相容問題或能升級才會啟用。

已知問題

以下列出部分 Firefox 3 已知問題，請閱讀後再回報新的程式錯誤。

所有系統
  • 如果 Firefox 發生錯誤時 Flash 是開啟的，那麼錯誤回報器可能不會執行 (bug 422308)。 這個問題已經在 Flash 10 中修復。
  • 在「歷史收藏庫」中刪除一筆紀錄，然後使用「清除隱私資料」可能會導致 Firefox 發生錯誤。(bug 426275)
  • 一些透過不適用的安裝碼安裝的套件可能無法正確安裝。(詳情請見bug 406807)
  • 「工具>選項」中的「個人隱私>歷史>保存我的歷史最少…天」，輸入 "0" 沒有反應。(bug 366075)
  • 某些網頁(像是 mlb.com)無法正確的偵測「Silverlight」是否被安裝，並且無法執行功能 ( bug 432371 )。這個問題已經在 Silverlight 2.0 beta 中修復。
  • DOM 觀查器已被移除，以額外的擴充套件形式提供。
  • Cross-Site XmlHttpRequest 支援暫時被移除。等到其規格更穩定、安全模組改進之後才會再加入。(bug 424923)
  • 當離開「你想要Firefox記住這個密碼嗎?」的提示後，有時候會出現除非重新載入頁面，不然會無法點擊表單的情形。(bug 433942)
  • 當使用 Window-Eyes 時，新的網址列、加入書籤對話框與下載管理員通知的訊息不一致，請使用 JAWS 8.0 或其他螢幕閱讀程式來測試 Firefox 3 (bug 393398)。 這個問題已經在新版的 Window-Eyes 中修復。
Windows 及 Windows Vista
  • Windows Vista 及某些其他版本Windows並沒有提供一個Windows Media Player (WMP) 的plugin。必須透過一些步驟才能安裝plugin及開啟Windows Media檔案。安裝完畢後可能也需要做Windows Updates才能讓plugin正常地顯示這些檔案。
  • 舊有的 Google Desktop Search 可能會造成 Firefox 在啟動時造成當機；重新安裝 Google Desktop Search 可以解決這個問題。(bug 401513)
Mac OS X
  • 如果您正利用 IPv6 從一個不支援 IPv6 路由偵測的網路位置，這可能會變得非常慢。您可以透過將 "network.dns.disableIPv6" 設為「否」作為作為替代辦法。(bug 417689)
Linux 及 Unix
  • Ubuntu 7.10 的使用者需要更新認證才能送出當機報告 (bug 407748)
  • 多功能滑鼠中，控制上、下頁的按鍵必須重新設定 (bug 420294)
  • NVIDIA 驅動程式與某些版本的 X server 的不相容可能會導致灰階圖片顯示不正確。 ( bug 411831 )
  • Orca 螢幕朗讀軟體無法閱讀某些文字，像是儲存密碼的通告、網頁個人隱私資料、或者是搜尋欄中預設的文字－這三點是 Orca 已知的問題 (詳情請見Gnome bug 533109 以及 533125 ) 這個問題已經在新版的 Orca 中修復。

疑難排解

  • 設計不良或不相容的套件將導致瀏覽器發生問題，發生的問題包含當機或網頁瀏覽速度變慢等等。如果您碰上瀏覽器無法開啟、視窗顯示不正常等等狀況，便可能是擴充套件或佈景主題的問題。這時，請將瀏覽器重新啟動在安全模式：
  • 在 Windows 下用「Safe Mode」捷徑或在命令列中輸入 firefox.exe -safe-mode 指令。
  • 在 Linux 下請執行./firefox -safe-mode 指令。
  • 在 MacOS X 下請輸入

    cd /Applications/Firefox.app/Contents/MacOS/
    ./firefox-bin -safe-mode

    當啟動安全模式時，所有的附加元件將停用並使用預設的佈景主題。在安全模式下移除有問題的擴充套件/佈景主題，然後再正常的啟動瀏覽器。
  • 如果您想在移除某擴充套件後，以 -install-global-extension 指令將同一擴充套件裝到每個使用者設定檔中，則必須在重新安裝前先重開瀏覽器一次，以便徹底移除擴充套件。如果您沒這麼做，則擴充套件清單可能會有衝突項目而不能為所有設定檔安裝。
  • 如果您遇到的怪問題與書籤、下載、視窗位置、工具列、瀏覽記錄或其他設定項目有關，建議您在回報問題前先行建立新的使用者設定檔，並查看能否重現該問題。建立新設定檔的方式如下：執行 Firefox 時加上 -P 的命令列參數，選擇「管理使用者設定檔」按鈕，然後選擇「建立使用者設定檔」。將各種設定(書籤、儲存的密碼等)從舊的設定檔中分次轉移到新的設定檔，每移轉一項便檢查是否發生問題。如果您發現某特定設定檔資料會造成問題，請回報錯誤並且附上該檔案。

常見問題集

  1. 我該怎麼幫忙呢？

    我們需要開發員和測試社群回饋建議，幫我們將 Firefox 變得更盡善盡美。請先閱讀錯誤回報注意事項，再到 Bugzilla 回報。您也可以使用回饋表單表達意見。

  2. 你們為什麼都沒有回信？

    請使用討論區， Firefox 團隊會定期閱讀上面的文章。我們常常都收到大量的信件，而您的可能不小心漏失了。

  3. 我該去哪裡取得新的佈景主題與擴充功能 (附加元件)？

    擴充套件及佈景主題可自 Mozilla 附加元件中心下載。

  4. Firefox 3 是哪些人做出來的？

    很多人。從「說明＞關於 Mozilla Firefox」中有份名單，列出部分曾對 Firefox 3 有所貢獻的人。

  5. Firefox 3 的原始碼在何處？

    Firefox 3.0.19 原始碼已開放下載，而目前開發中的最新程式碼則需以 CVS 取得。Firefox 相關程式為「mozilla/browser」、「mozilla/toolkit」及「mozilla/chrome」，請參考建製步驟。

  6. 電子郵件程式呢？

    Firefox 3能與您預設的任何郵件程式協同運作， 而我們推薦您試試我們下一代的郵件軟體：與 Firefox 合作無間的 Mozilla Thunderbird。

非官方建製版

Mozilla 代表 各國作者提供了各國語系的地方化版本，您可至Firefox 3 下載頁取得檔案。

其他尚未被官方 Firefox 3 驗證通過的本地化版本可以在 官方站台取得。
