Firefox 新鮮事

  • 全新功能

    右鍵選單中新增「複製不含追蹤碼的鏈結」功能，確保複製的連結不包含追蹤資訊。

  • 全新功能

    Firefox 「偏好設定 → 隱私權與安全性」中新增 全球隱私控制 (Global Privacy Control, GPC) 選項。透過這項自願勾選的功能，Firefox 會告知網站「使用者不希望其資料被共享或出售」。此功能在隱私瀏覽模式下預設啟用。

  • 全新功能

    Firefox 的隱私視窗和「加強型追蹤保護」的「嚴格」模式，現在強化了 Canvas API 的指紋保護，以持續保護我們使用者的線上隱私。

  • 全新功能

    在接下來的幾週內，Firefox 將會在德國使用者的隱私視窗中預設啟動「Cookie 橫幅阻擋器」（Cookie Banner Blocker）。Firefox 現在將自動拒絕 cookie，並關閉支援網站上討人厭的 cookie 橫幅。

  • 全新功能

    現在德國所有使用者的隱私視窗中預設啟用 URL 追蹤保護。Firefox 將移除經常用來追蹤使用者的非必要 URL 查詢參數。

  • 全新功能

    Firefox 現在會從作業系統的根憑證庫匯入 TLS 信任錨點（例如憑證）。此功能在 Windows、macOS 和 Android 上預設啟用，如有需要的話，可在設定中關閉（偏好設定 → 隱私權與安全性 → 憑證）。

  • 全新功能

    about:logins 中新增了編輯和刪除選定憑證的鍵盤快速鍵。編輯為 Alt + enter（macOS 上為 Option + return），刪除為 Alt + Backspace（macOS 上為 Option + Delete）。

  • 全新功能

    在 Ubuntu Linux 上，當 Chromium 和 Firefox 皆透過 Snap 安裝時，即可從 Chromium 匯入資料。

  • 全新功能

    子母畫面在 Windows 和 Linux 上支援角落吸附功能－只需在移動子母畫面時按住 Ctrl 鍵。

  • 修正

    多項 安全性修正。

  • 企業版

    你可以在 Firefox 120 企業版版本資訊 中找到政策更新和企業版相關的錯誤修復資訊。

  • 開發功能

    Firefox 120 開發者資訊

  • 開發功能

    新增 User Activation API，使 JavaScript 可透過 navigator.userActivation 檢查使用者目前或是否曾經與頁面產生互動（如點擊等）。

  • 開發功能

    啟用「Early Hints Preconnect」（預先連線的提前提示），支援提供資訊的 103 Early Hints 狀態碼，使伺服器能夠在最終 HTTP 回應之前先行傳送含所需資源連結的檔頭，為使用此功能的伺服器改善效能。（詳細資訊）

  • 開發功能

    使用者現在可以透過新增的開發者工具，來模擬瀏覽器分頁處於離線狀態。（網路 → 不限速 → Offline）

  • 開發功能

    樣式編輯器面板底部現新增了一個「Pretty print 樣式表」按鈕，類似於除錯器面板中的 Pretty Print 原始碼按鈕。可用於格式化樣式表（例如哪些被壓縮過的樣式表）。先前自動格式化壓縮文件的功能已移除。

  • 開發功能

    檢測器面板中的規則面板，不再將新的 CSS Color 4 顏色（例如 OKLCH）顯示為十六進位／命名顏色，以確保它與原來使用的值相符。

  • 網路平台

    現在可以正確解析和計算 lh 及 rlh 單位為 長度，讓作者能根據元素（或根元素）的行高來指定長度。

  • 網路平台

    現已預設啟用 WebAssembly GC，允許新的語言──如 Dart 或 Kotlin──在 Firefox 上運行。這使得客戶語言和主瀏覽器之間的參考循環可以被回收。

下載與安裝

系統需求

在安裝 Firefox 前請先確定您的電腦是否合乎系統需求。

下載 Firefox

您可以由此下載最新的 Firefox，或取得 Windows、Linux 及 Mac OS X 平台上的各國語言版本，或由此下載 Firefox 未來發行版本（測試版）。若需自行編譯其他平台或語言的版本，請見本文件最末節。

安裝 Firefox

請注意：安裝新版 Firefox 將覆蓋現存的 Firefox 程式，您不會遺失書籤或瀏覽記錄，但有些附加元件可能要更新才能繼續使用。詳細安裝步驟請參見如何安裝與更新 Firefox。

移除 Firefox

在 Windows 作業系統中，您可以使用「新增或移除程式」移除 Firefox。macOS 的使用者請將 Firefox 程式移至垃圾筒；Linux 使用者則請移除包含 Firefox 的資料夾。

一般狀況下，移除 Firefox 時不會把書籤、瀏覽記錄、擴充套件和其他附加元件一併刪除，這些資料會保留在您的「個人設定資料夾」內，此資料夾可以在「說明」選單的「更多疑難排解資訊」找到，按下「應用程式一般資訊」中「設定檔目錄」旁的按鈕便可開啟它。

請留意，若您沒有手動刪除個人設定資料夾，移除後重新安裝 Firefox 將繼續沿用原本的設定檔。

附加元件及佈景主題

您可以由「附加元件管理員」或附加元件網站取得附加元件及佈景主題。 當 Firefox 升級時，附加元件可能因不相容於新版 Firefox 而被停用，此時則需要原作者進行升級。

疑難排解

  • 設計不良或不相容的套件將導致瀏覽器發生問題，包含當機或網頁瀏覽速度變慢等等狀況。如果您碰上瀏覽器無法開啟、視窗顯示不正常等等狀況，便可能是擴充套件或佈景主題的問題。這時請參考「Firefox 輔助說明：安全模式」一文開啟 Firefox 安全模式。在安全模式下，所有附加元件及佈景主題將被停用。請您在安全模式下移除有問題的擴充套件或佈景主題，再正常地啟動瀏覽器。
  • 如果您遇到與書籤、下載、視窗位置、工具列、瀏覽記錄或其他奇怪問題，建議您在回報問題前先建立新的使用者設定檔試試。請參考「Firefox 輔助說明：使用設定檔管理員建立或移除 Firefox 設定檔」一文建立新的使用者設定檔，並將各種設定（書籤、儲存的密碼等）從舊的設定檔中分次轉移到新的設定檔（可透過 Firefox Sync 同步功能進行）。

常見問題集

  1. 我該怎麼幫忙呢？

    我們需要開發員和測試社群回饋建議，幫我們將 Firefox 變得更盡善盡美。請先查閱 Firefox 輔助說明看看有沒有相關解答，再到 Bugzilla 回報你發現的錯誤；您也可以使用意見回饋功能表達意見。

  2. 我如何預覽未來的 Firefox 版本？

    加入 Aurora 或 Beta 開發頻道就可以搶先試用 Firefox 開發中版本。

  3. 我該到哪裡才能用中文問問題跟表示意見？

    請使用 Firefox 討論區，MozTW 社群團隊與熱心的 Firefox 用戶會定期閱讀上面的文章。

  4. 我該去哪裡取得新的佈景主題與擴充功能 (附加元件)？

    請到 Mozilla 附加元件中心下載 Firefox 的擴充套件及佈景主題。

  5. Firefox 是哪些人做出來的？

    Firefox 是由世界各地的 Mozilla 社群成員與員工合力開發。這裡有份長長的名單，列出部分曾對 Firefox 有重要貢獻的人。想要加入我們的大家庭嗎？或者加入 Mozilla 台灣社群？

  6. Firefox 的原始碼在何處？

    請參照 Getting Mozilla Source Code Using Mercurial 一文取得 Firefox 原始碼；如需編譯 Firefox，請參考 Build Instructions。

  7. 電子郵件程式呢？

    Firefox 能與您預設的任何郵件程式協同運作，而我們推薦您試試我們下一代的郵件軟體：與 Firefox 合作無間的 Mozilla Thunderbird。
