Firefox 新鮮事

  • 2023 年的 1 月，Microsoft 終止了對 Windows 7 以及 Windows 8 的支援；因此，這個 Firefox 版本將是這些作業系統使用者最後能收到的版本。Windows 7 和 Windows 8 的使用者將會自動轉移到 Firefox ESR 115 版，以便繼續接收重要的安全更新。深入資訊可參閱本 Firefox 支援文件。

    同樣地，這個 Firefox 版本也會是最後一個支援 Apple macOS 10.12、10.13 和 10.14 的主要版本。這些作業系統的使用者將會自動轉移到 Firefox 的 ESR 115 版本，使這些使用者依然可以接收重要的安全性更新。深入資訊可參閱本 Firefox 支援文件。

    這裡我們想特別感謝所有為此次 Firefox 發行版做出貢獻的新摩茲人！

  • 全新功能

    打算從其他瀏覽器遷移過來嗎？現在可以將 Chrome 基礎的瀏覽器中保存的付款方式帶到 Firefox。

  • 全新功能

    現在在 Intel GPU 的 Linux 上啟用了硬體視訊解碼。

  • 全新功能

    「分頁管理器」的下拉選單現已加入關閉按鈕，讓關閉分頁的動作更為簡便。

  • 全新功能

    重整及簡化從其他瀏覽器匯入資料的使用者介面。

  • 全新功能

    平台不支援 H264 視訊解碼的使用者，現在可以回退使用 Cisco 的 OpenH264 外掛程式來播放影片。

  • 修正

    Firefox 標題列可見時，Windows 放大鏡現在會正確跟著文字游標移動。

  • 修正

    使用低階 Wi-Fi 或 USB Wi-Fi 驅動程式，且停用作業系統地理定位功能的 Windows 使用者，現在可以根據情況個別核核准位置定位，不再會導致整個系統網路不穩。

  • 修正

    修補多個安全弱點。

  • 變更

    密碼輸入欄位現已支援「復原」與「取消復原」。

  • 變更

    Linux 上，現在對「新增分頁」按鈕點下滑鼠中鍵，會在新分頁打開 xclipboard 的內容。如果 xclipboard 的內容是網址，則會開啟這個網址；如果是其他文字，則會使用您預設使用的搜尋提供者進行搜尋。

  • 變更

    對於使用 Firefox Colorways 內建佈景的使用者，如果 Firefox 設定檔停用了擴充套件自動更新，這個佈景主題將會自動轉換到 addons.mozilla.org 上的相同佈景主題。如此稍後當 Firefox 安裝程式移除 Colorways 後，使用者將能繼續保留 Colorways 主題。

  • 變更

    某些 Firefox 使用者可能會在擴充套件面板看到訊息，指出「使用者的擴充套件不允許在當前開啟的網站上使用」。出自各種原因（包含安全考慮），我們引入了一個新的後端功能，在特定網站上只允許某些受 Mozilla 監管的擴充套件執行。

  • 企業版

    在最新的 Firefox 中修復多個 Bug 與加入新政策，您可在 Firefox 115 企業版版本資訊之中找到更多資訊。

  • 開發功能

    Firefox 115 開發者資訊

  • 開發功能

    網站開發時，我們會依賴一些除錯時你不會在意的第三方函式庫。這些函式庫現在可以設為忽略（ignore）了。「忽略」代表遇到斷點時不停下，並且會在逐步執行時被略過。

    你現在可以選擇「隱藏忽略的原始碼」，以隱藏開發者工具的原始碼樹狀圖中那些已忽略的項目 (bug)。

    「JavaScript 追蹤」（JavaScript tracing）功能也會略過忽略的來源。 (bug)

  • 開發功能

    我們加入了一個新選項 devtools.f12_enabled，以防止因意外按到 F12 鍵而開啟開發者工具。(bug)

  • 網路平台

    內建編輯器分割與結合節點的行為，已經和其他使用 contenteditable 和 designMode 的瀏覽器行為對齊。例如，按下 Enter 鍵來分割段落、或在段落開頭按下 Backspace 將段落和前一個段落合併。

    當一個節點被分割時，內建編輯器會在原本的節點後方建立新節點（而不是在前方）。換句話說：節點會建立在右邊而不是左邊。

    類似的，合併兩個節點時，內建編輯器會刪除後面的節點，並將其子節點移到前面節點的結尾，而不是刪除前面的節點，並將子節點移到後面結尾的開頭。

  • 網路平台

    WebRTC 應用程式開發者現在可以指定抖動緩衝區（jitter buffer）保持的毫秒目標值，允許應用程式在播放延遲及（由於網路抖動而）用光音訊或視訊影格之間進行權衡。

  • 網路平台

    變動陣列時回傳新副本（change array by copy），Array.prototype 和 TypedArray.prototype 提供了額外的方法，可以在變更陣列時以回傳新複本的模式來進行。

  • 網路平台

    現已支援 animation-composition 屬性，當多個動畫效果同時影響同個屬性時，提供一種宣告方式以定義應使用何種合成行為。

  • 網路平台

    加入 URL.canParse() 函式，可以更快、更簡便地檢查 URL 是否有效且可以解析。

  • 網路平台

    多虧在磁碟上加密儲存的方式，在隱私瀏覽現已支援不受記憶體限制的 IndexedDB。用來解密資料的暫時性金鑰只存在記憶體中，且所有儲存的資訊將會在隱私瀏覽工作階段正常結束時從磁碟清空。

  • 網路平台

    CSS 導入規則（CSS import rules）現在支援條件式規則（@import supports(...)）

  • 請參考此版本的完整變更清單。 您可能也想瞭解前一個版本的更新。

下載與安裝

系統需求

在安裝 Firefox 前請先確定您的電腦是否合乎系統需求。

下載 Firefox

您可以由此下載最新的 Firefox，或取得 Windows、Linux 及 Mac OS X 平台上的各國語言版本，或由此下載 Firefox 未來發行版本（測試版）。若需自行編譯其他平台或語言的版本，請見本文件最末節。

安裝 Firefox

請注意：安裝新版 Firefox 將覆蓋現存的 Firefox 程式，您不會遺失書籤或瀏覽記錄，但有些附加元件可能要更新才能繼續使用。詳細安裝步驟請參見如何安裝與更新 Firefox。

移除 Firefox

在 Windows 作業系統中，您可以使用「新增或移除程式」移除 Firefox。macOS 的使用者請將 Firefox 程式移至垃圾筒；Linux 使用者則請移除包含 Firefox 的資料夾。

一般狀況下，移除 Firefox 時不會把書籤、瀏覽記錄、擴充套件和其他附加元件一併刪除，這些資料會保留在您的「個人設定資料夾」內，此資料夾可以在「說明」選單的「更多疑難排解資訊」找到，按下「應用程式一般資訊」中「設定檔目錄」旁的按鈕便可開啟它。

請留意，若您沒有手動刪除個人設定資料夾，移除後重新安裝 Firefox 將繼續沿用原本的設定檔。

附加元件及佈景主題

您可以由「附加元件管理員」或附加元件網站取得附加元件及佈景主題。 當 Firefox 升級時，附加元件可能因不相容於新版 Firefox 而被停用，此時則需要原作者進行升級。

疑難排解

  • 設計不良或不相容的套件將導致瀏覽器發生問題，包含當機或網頁瀏覽速度變慢等等狀況。如果您碰上瀏覽器無法開啟、視窗顯示不正常等等狀況，便可能是擴充套件或佈景主題的問題。這時請參考「Firefox 輔助說明：安全模式」一文開啟 Firefox 安全模式。在安全模式下，所有附加元件及佈景主題將被停用。請您在安全模式下移除有問題的擴充套件或佈景主題，再正常地啟動瀏覽器。
  • 如果您遇到與書籤、下載、視窗位置、工具列、瀏覽記錄或其他奇怪問題，建議您在回報問題前先建立新的使用者設定檔試試。請參考「Firefox 輔助說明：使用設定檔管理員建立或移除 Firefox 設定檔」一文建立新的使用者設定檔，並將各種設定（書籤、儲存的密碼等）從舊的設定檔中分次轉移到新的設定檔（可透過 Firefox Sync 同步功能進行）。

常見問題集

  1. 我該怎麼幫忙呢？

    我們需要開發員和測試社群回饋建議，幫我們將 Firefox 變得更盡善盡美。請先查閱 Firefox 輔助說明看看有沒有相關解答，再到 Bugzilla 回報你發現的錯誤；您也可以使用意見回饋功能表達意見。

  2. 我如何預覽未來的 Firefox 版本？

    加入 Aurora 或 Beta 開發頻道就可以搶先試用 Firefox 開發中版本。

  3. 我該到哪裡才能用中文問問題跟表示意見？

    請使用 Firefox 討論區，MozTW 社群團隊與熱心的 Firefox 用戶會定期閱讀上面的文章。

  4. 我該去哪裡取得新的佈景主題與擴充功能 (附加元件)？

    請到 Mozilla 附加元件中心下載 Firefox 的擴充套件及佈景主題。

  5. Firefox 是哪些人做出來的？

    Firefox 是由世界各地的 Mozilla 社群成員與員工合力開發。這裡有份長長的名單，列出部分曾對 Firefox 有重要貢獻的人。想要加入我們的大家庭嗎？或者加入 Mozilla 台灣社群？

  6. Firefox 的原始碼在何處？

    請參照 Getting Mozilla Source Code Using Mercurial 一文取得 Firefox 原始碼；如需編譯 Firefox，請參考 Build Instructions。

  7. 電子郵件程式呢？

    Firefox 能與您預設的任何郵件程式協同運作，而我們推薦您試試我們下一代的郵件軟體：與 Firefox 合作無間的 Mozilla Thunderbird。
