Firefox 是速度快、功能又齊全的瀏覽器，它讓上網比以往更有效率。其它關於 Firefox 的訊息，請參考 Firefox 產品首頁。

此版本資訊包含了 Firefox 1.0 的新鮮事、下載與安裝指示、已知問題以及常見問題等訊息。如發現任何程式錯誤，請先行將版本資訊與錯誤回報注意事項閱讀完畢，再到 Bugzilla 回報。

我們想知道您對 Firefox 的意見。歡迎到討論區討論相關議題：您可至MozillaZine上的英文討論區或 MozillaTaiwan的Firefox 中文討論區與我們分享意見。

1.0.1 新鮮事

重要注意事項

如果您在 Firefox 的位址列輸入網址時 Firefox 常當掉，您可以先移除 Firefox 1.0.1、刪除 Firefox 程式資料夾（Windows 版可能會在 C:\Program Files\Mozilla Firefox 下、Linux 版則可能在 /home/<user>/firefox-installer/ 下）後重新安裝 Firefox 1.0.1，如此應可解決問題。

為避免這類問題發生，以安裝程式安裝新版時，請不要將程式裝到舊 Zip 版本的同一目錄中。

以下是 Firefox 1.0.1 版的新鮮事：

  • 提高穩定性。
  • International Domain Name（國際化域名，IDN）現在都會以 punycode 顯示，若想讓 IDN 以 Unicode 顯示，請將「network.IDN_show_punycode」設定為 false。
  • 修復一些安全性問題：
    • MFSA 2005-29 Internationalized Domain Name (IDN) homograph spoofing
    • MFSA 2005-28 Unsafe /tmp/plugtmp directory exploitable to erase user's files
    • MFSA 2005-27 Plugins can be used to load privileged content
    • MFSA 2005-26 Cross-site scripting by dropping javascript: link on tab
    • MFSA 2005-25 Image drag and drop executable spoofing
    • MFSA 2005-24 HTTP auth prompt tab spoofing
    • MFSA 2005-23 Download dialog source spoofing
    • MFSA 2005-22 Download dialog spoofing using Content-Disposition header
    • MFSA 2005-21 Overwrite arbitrary files downloading .lnk twice
    • MFSA 2005-20 XSLT can include stylesheets from arbitrary hosts
    • MFSA 2005-19 Autocomplete data leak
    • MFSA 2005-18 Memory overwrite in string library
    • MFSA 2005-17 Install source spoofing with user:pass@host
    • MFSA 2005-16 Spoofing download and security dialogs with overlapping windows
    • MFSA 2005-15 Heap overflow possible in UTF8 to Unicode conversion
    • MFSA 2005-14 SSL "secure site" indicator spoofing
    • MFSA 2005-13 Window Injection Spoofing

1.0 新鮮事

以下是 Firefox 1.0 版的新鮮事：

  • 更棒的分頁瀏覽控制方式

下載與安裝

系統需求

在安裝 Firefox 前請先確定您的電腦合乎系統需求。

下載 Firefox

Mozilla.org 提供了 Windows、Linux 及 Mac OS X 平台上數十種語言版的 Firefox。欲下載最新的 Firefox，請到 GetFirefox.com 或瀏覽 FTP。 其他不是由 Mozilla.org 官方提供的平台及語言版本，請見本文件最後的「社群貢獻版」一節

安裝 Firefox

下載 Firefox 安裝程式或壓縮檔後，請依下列步驟進行安裝：

Windows

以滑鼠左鍵雙點 Firefox Setup 1.0.exe (中文版應為 firefox-1.0.zh-TW.win32.installer.exe) 安裝程式圖示以開始安裝。

MacOS X

以滑鼠雙點 Firefox 壓縮磁碟映像檔圖示以便裝載(mount)， 則瀏覽器便應能自動解壓縮並安裝完畢。再以滑鼠雙點解壓後的 Firefox 磁碟映像圖示，然後將 Firefox 應用程式拖曳到您的磁碟。如想讓圖示在 Dock 上顯示，可以自行將 Firefox 圖示拖曳至其上。

Linux/GTK2

解壓縮後以下列方式執行安裝程式：

tar -xzvf firefox-1.0.zh-TW.linux-i686.installer.tar.gz
cd firefox-1.0.zh-TW.linux-i686.installer.tar.gz
./firefox-installer

如果您有 Nautilus 能執行 Executable Text File，那麼只要雙點 firefox-installer 即可。

其它平台

將壓縮檔解壓縮後執行 firefox

移除 Firefox

欲移除 Firefox，請依下列步驟操作：

Windows

選擇「開始＞控制台」，當控制台出現時，請以滑鼠左鍵雙點「新增或移除程式」， 然後在清單中找到「Mozilla Firefox (1.0)」並按下「移除」。

MacOS X

將 Firefox 程式拖曳至垃圾桶。

其他平台

移除 firefox 資料夾。

上述步驟將保留您的個人設定檔，供日後重新安裝時使用。如果您想移除 Firefox 的個人設定檔資料夾，請依循「個人設定資料夾」一節的步驟。請注意：這麼一來您將失去所有書籤、儲存密碼、設定及其他資訊，重新安裝 Firefox 時就必須重新建立個人設定。

 

其他資訊

個人設定資料夾

Firefox 將您的個人設定檔存放在下列位置之一：

Windows 2000、XP Documents and Settings\<UserName>\Application Data\Mozilla\Firefox
Windows NT WINNT\Profiles\<UserName>\Application Data\Mozilla\Firefox
Windows 98、ME Windows\Application Data\Mozilla\Firefox
MacOS X ~/Library/Application Support/Firefox
Linux 及 Unix 系統 ~/.mozilla/firefox

如果您在升級至 Firefox 1.0PR 前將 Firefox 0.8 設為預設瀏覽器，那麼您的設定檔將被複製至新的位置， 您可自行決定是否將舊的「Phoenix」刪除。

擴充套件及佈景主題

當您升級到新版的 Firefox 時，所有的擴充套件及佈景主題會自動停用，直到 Firefox 發現 1) 已安裝的套件可用於新版 或 2)有新版套件可相容新版 Firefox。這是因為 Firefox 每一版都有所更新，與擴充套件的整合模式也不盡相同，如果使用舊版套件反而可能發生問題。如果您喜歡的擴充套件或佈景主題未能相容新版，請寫信給作者，鼓勵作者更新。

若您自 0.8 或更早的版本升級，則您首次啟動 Firefox 1.0 時，Firefox 無法自動查閱擴充套件及佈景主題是否有更新相容版本，請自行尋找並安裝相容新版。

在 0.8 版時部分使用者曾因「No XBL Binding for Browser」錯誤訊息而無法啟動 Firefox，並隨即發現佈景主題失效。為了避免這種問題，在您改用 Firefox 1.0 時，所有的擴充套件將自動停用。如果您曾使用 0.9 或 1.0PR 版，則當您首次啟動 1.0 時，Firefox 會試著連上 update.mozilla.org 查閱擴充套件是否有更新的相容版本。如果有，則會下載並安裝之。如果您用的是 0.8 版，則得請您自行尋找相容新版。

其他變動

  • Firefox 現在於 Linux 上遵照 GTK 於文字輸入欄的鍵盤設定，所以 Ctrl+字母 為程式快速鍵而非 readline-like text-editing shortcuts. 請參考 這份文件以獲得其他資訊。

 

已知問題

以下列出 Firefox 1.0 的部分已知問題， 請閱讀後再回報新的程式錯誤，並請定期檢查本欄的更新內容﹝如在此版本發現新問題，我們會隨時在英文網頁更新此欄﹞。

所有系統

  • 如果您將 Firefox 安裝於權限有所管制的區域，那麼就得先由擁有權限的使用者啟動一次 Firefox，以便建立應有的初始啟動檔案。若由權限不足的使用者先行啟動，則可能會因為無法完整建立初始啟動檔案而不斷重新啟動。
  • 當您升級時，所有的擴充套件及佈景主題都將停用。這其實不算是「問題」，但可能有人會問（所以就列在這裡。）原理請參考前述「擴充套件及佈景主題」一節。
  • 軟體升級程序不會進行 Proxy 驗證，所以如果您需透過 Proxy 伺服器上網，自動更新將無法進行。（錯誤資訊）您可先於瀏覽器中查閱網頁，藉此進行驗證後再行升級。
  • 若您未擁有 Firefox 安裝處的檔案寫入權限，軟體升級程序便無法進行，這是因為軟體升級必須於安裝處替換或建立檔案。
  • 關於「Single Window Mode」（單視窗模式）選項的說明文件中提到可以「強制令視窗開於新分頁或同分頁」。這個功能最後在推出前取消了，因為我們碰到一些問題暫無法解決，所以請跳過那一段。如果想重新啟用單視窗模式，可以使用設定控制台（在位址列輸入「about:config」然後按下 Enter），將 browser.tabs.showSingleWindowModePrefs 設定為 true。重新啟用此功能可能造成當機等問題，請自負責任。

Windows

  • 以權力受限的一般使用者身份安裝 Firefox 時，就算安裝在可寫入的地方，有時依然會發生些許問題（像是預設瀏覽器設定錯誤等），但瀏覽器還是可以用。當以受限使用者身份安裝時，請不要在未移除舊版的情況下指定安裝於同一資料夾，否則可能無效。
  • 若使用某些舊版的 Adobe Acrobat Reader plug in，閱讀 PDF 後 Firefox 可能會當掉。如果您碰到這種問題，請到 Adobe 網站下載 新版 Plug in。
  • 在 Windows 98 系統上，應用程式的圖示可能會顯示為 Windows 圖示。

MacOS X

  • MacOSX 使用者:不要由磁碟映像中啟動 Firefox！這麼一來 Firefox 可能會不斷重新啟動（症狀是 Firefox 圖示在 Dock 中不斷跳起、消失再出現）。若要從不斷啟動的情形中終止 Firefox，請開啟 終端機（Terminal）後輸入「killall firefox-bin」並按下 Enter。之後請將 Firefox 安裝於有權寫入的位置、再試一次。若安裝於多人共用的電腦上，可以管理員身份安裝於共用位置，執行一次後所有使用者就都能用了。
  • 如果 Firefox 未顯示瀏覽器視窗，請按 Cmd+Q 離開 Firefox，然後開啟 ~/Library/Application Support/Firefox/Profiles/<garble>.default/ 資料夾並刪除 localstore.rdf 檔，再重新啟動 Firefox。您之前的自訂的工具列及視窗位置等設定將因而消失。

Linux 及 Unix 系統

  • 若 Firefox 安裝的路徑中含有空白字元，則可能無法設定為預設瀏覽器。這樣每次啟動時變會不斷詢問「是否要設為預設瀏覽器」。將 Firefox 安裝在不含空白字元的路徑就不會有這個問題。
  • 在Fedora Core 3(FC3)中，GNOME integration並不能適當的運作。FC3的使用者需要安裝linc-1.0.3-3.1.i386.rpm。安裝好這個RPM後，在您安裝Firefox的目錄下執行以下的命令(您需要有寫入的權限)：

    touch .autoreg

    下一次您啟動Firefox時，GNOME integration將會正常的運作。

Web Page Rendering

  • Firefox 與其他 Mozilla 產品都使用相同的 Gecko 佈局引擎。如果您發現有某個網站無法正確顯示，那大多屬於 Gecko 的問題，而與 Firefox 本身無甚牽連。這些問題應於 Bugzilla 中提報為 Browser 產品類錯誤，而非提報為 Firefox 錯誤。如果您對技術小有心得，請試著找出引發問題的原因，並製作測試網頁以便更精確描述問題所在。

欲取得其他問題的訊息、常見問題集、使用技巧以及一般 Firefox 協助，請到 David Tenser 非常有用的 Firefox Help網站、MozillaZine的Firefox 英文討論區或林弘德所維護的Firefox 中文討論區。

設定控制台（在網址列輸入「about:config」後按 Enter 時出現的東西）提供 Firefox 進階操作控制，可以直接修改 Firefox 設定。這些數值若未善加修改，可能會毀了 Firefox！請再修改前閱讀說明文件，以便得知相關數值如何調整。

疑難排解

  • 若您在昇級瀏覽器前，先停用先舊版的擴充套件，那麼在昇級時便不需建立新的設定檔。如欲停用擴充套件，請先開啟舊版瀏覽器、點選工具列上的工具／擴充套件、在擴充套件列表項目上按右鍵，最後選停用即可。現在您可以放心安裝新版 Firefox 了。
  • 不良的套件設計或不相容的套件將導致瀏覽器發生問題，發生的問題包含當機或網頁瀏覽速度變慢等等。如果您遇到瀏覽器無法開啟、視窗顯示不正常等等奇怪的問題，您可能是遇到擴充套件或佈景主題的問題。將瀏覽器重新啟動在安全模式。在Windows下用"Safe Mode"捷徑或在命令列下firefox.exe -safe-mode指令。在Linux下執行./firefox -safe-mode指令。在MacOS X下

    cd /Applications/Firefox.app/Contents/MacOS/
    ./firefox-bin -safe-mode

    當啟動安全模式時，所有的擴充套件將停止使用，並使用預設的佈景主題。在安全模式下移除有問題的擴充套件/佈景主題，然後再正常的啟動瀏覽器。

如果您想在移除某擴充套件後，以 -install-global-extension 指令將同一擴充套件裝到每個使用者設定檔中，則必須在重新安裝前先重開瀏覽器一次，以便徹底移除擴充套件。如果您沒這麼做，則擴充套件清單可能會有衝突項目而不能為所有設定檔安裝。

如果您遇到的怪問題與書籤、下載、視窗位置、工具列、瀏覽記錄或其他設定項目有關，建議您在回報問題前先行建立新的使用者設定檔，並查看能否重現該問題。建立新設定檔的方式如下：執行 Firefox 時加上 -P 的命令列參數，選擇「管理使用者設定檔」按鈕，然後選擇「建立使用者設定檔」。將各種設定(書籤、儲存的密碼等)從舊的設定檔中分次轉移到新的設定檔，每移轉一項便檢查是否發生問題。如果您發現某特定設定檔資料會造成問題，請回報錯誤並且附上該檔案。

 

常見問題集

  1. 我該怎麼幫忙呢？

    我們需要大量曝光。讓推廣 Mozilla Firefox 成為您的任務，使您的親朋好友上司下屬都認識它： 如果您是個學生，就讓它廣佈校園之中；寫篇文章投稿至Slashdot等新聞站台上，發布 Firefox 新版的新聞；也可以在您的 Blog 上透漏一點相關訊息； 總之，儘可能的在網際網路中散佈，就像 Kazaa 的威力一樣，處處推廣 Firefox！

  2. 我使用其他瀏覽器，因為…

    如果您不願使用 Firefox，請告訴開發團隊您的理由。我們會持續注意Firefox forums中的使用者意見。

  3. 你們為什麼都沒有回信？

    請使用討論區， Firefox 團隊會定期閱讀上面的文章。我們常常都收到大量的信件，而您的可能不小心漏失了。

  4. 我該去哪裡取得新的佈景主題與擴充功能？

    擴充套件及佈景主題可自Mozilla Update下載。

  5. 哪些人正致力於 Firefox 的開發工作？

    很多人。詳細列表請參考「說明＞關於 Firefox」

  6. Firefox 的原始碼在何處？

    cvs.mozilla.org. Mozilla trunk + mozilla/browser + mozilla/toolkit.

  7. 電子郵件程式呢？

    Firefox 能與您預設的任何郵件程式協同運作， 而我們推薦您試試我們下一代的郵件軟體：與 Firefox 合作無間的 Mozilla Thunderbird。
