版本資訊 - Firefox 0.9 (One Tree Hill)

目前版本 0.9.3 版

Firefox 是速度快、功能又齊全的瀏覽器，它讓上網比以往更有效率。其它關於 Firefox 的訊息，請參考[Firefox Help]。

Firefox 0.9 為「技術預覽」版。即使本軟體性能良好，在大多數情形下足以成為您的主要瀏覽器，但我們不對其性能與可靠性做任何保證。這是尚未完工的版本，不應在重要關鍵作業上使用。詳情請見 [使用同意書]。

此版本資訊包含了 Firefox 0.9 版的新鮮事、下載與安裝指示、已知問題以及常見問題集等訊息。如發現任何程式錯誤，請先行閱讀這份訊息與[錯誤回報注意事項]，再到 Bugzilla 回報。

我們想知道您對 Firefox 的回應。請到討論區與我們分享意見：您可至 [MozillaZine] 上的[英文討論區]或 [Mozilla@Taiwan] 的 [Firefox 中文討論區]與我們分享意見。

新鮮事

以下是 Firefox 此版的新鮮事：

全新預設佈景主題
更新的預設佈景主題現在提供三種平台上統一的視覺，對 Windows 的使用者來說更是一個清爽的外表。GNOME 的微調將在未來版本跟進。
能轉移多種設定資料
現在改用 Firefox 更是容易，讓您輕易將 Internet Explorer 中的「我的最愛」、「記錄」、選項設定、Cookies 及密碼匯入 Firefox 中。Firefox 也能匯入 Mozilla 1.x、Netscape 4.x、6.x、7.x 及 Opera 的設定。MacOS X 及 Linux 平台的 Safari、OmniWeb、Konqueror 等瀏覽器也將在未來版本加以支援。
擴充套件/佈景主題管理員
嶄新的擴充套件及佈景主題管理員，讓您更方便管理及更新這些附加套件。SmartUpdate 也同時能在 Firefox 有所更新時提醒您。
檔案大小再瘦身
Windows 的使用者將發現 Firefox 現在只有 4.6 MB 的大小，容易下載。
說明系統
添加新的線上說明系統。
修訂錯誤、增強功能
複製圖片、從自動完成列表中刪除特定資料、在 GNOME 上透過 gnome-vfs 支援 SMB/SFTP 協定、更好的書籤、增強的搜尋以及其他精緻功能，讓您的瀏覽體驗更上層樓。

給 Linux/GTK2 的使用者

安裝程式
Firefox 現在提供 Linux/GTK2 上的安裝程式，讓使用者更容易安裝 Firefox。
視覺上的更新
我們不斷更新 Firefox 與您 GTK2 佈景主題的呼應方式 (像是選單的樣式)，而今已有成果。
GTK2 上的 Talkback 功能
請用此當機回報工具提交報告，幫助我們降低當機發生頻率。

請參考 [Burning Edge's Bigger Picture]，以獲取更多相關訊息。

安裝 Firefox

下載 Firefox 安裝程式或壓縮檔後，請依下列步驟進行安裝：

在安裝這些預覽版本之前，強烈建議您先移除所有曾安裝的舊版。將新版 Firefox 以覆蓋方式安裝於舊版目錄之上可能會發生意想不到的問題。如您直接覆蓋舊版，發生問題時請在回報前先將 Firefox 重新安裝於新的空目錄下，確認仍有相同錯誤後再行回報。

Windows

以滑鼠左鍵雙點 FirefoxSetup-0.9.3.exe (中文 0.9.1 版應為 Firefox-0.9.1-win32-zhTW.exe) 安裝程式圖示以開始安裝。

注意 - 請勿將 Firefox 直接以覆蓋舊版本的方式安裝。如果您想將 Firefox 0.9 安裝於 Firefox 0.8 的相同目錄，請先移除 Firefox 0.8。升級的問題會在日後的版本解決。

MacOS X

以滑鼠雙點 Firefox 壓縮磁碟影像檔圖示以進行裝設(mount)，則瀏覽器便應能自動解壓縮並安裝完畢。再以滑鼠雙點解壓後的 Firefox 0.9 磁碟影像圖示，然後將 Firefox 應用程式拖曳到您的磁碟。如想讓圖示在 Dock 上顯示，可以自行將 Firefox 圖示拖曳至其上。

Linux/GTK2

解壓縮後以下列方式執行安裝程式：

tar -xzvf firefox-0.9.3-i686-linux-gtk2+xft.tar.gz
cd firefox-0.9.3-i686-linux-gtk2+xft/
./firefox-installer

其它平台

將壓縮檔解壓縮後執行 firefox

移除 Firefox

欲移除 Firefox，請依下列步驟操作：

Windows

選擇「開始＞控制台」，當控制台出現時，請以滑鼠左鍵雙點「新增／移除程式」，然後在清單中找到「Mozilla Firefox」並按下「移除」。

MacOS X

將 Firefox 程式拖曳至垃圾桶。

其他平台

移除 firefox 資料夾。

上述步驟將保留您的個人設定檔，供日後重新安裝時使用。如果您想移除 Firefox 的[個人設定檔資料夾]，請依循以下步驟：

Windows

通常在 Documents and Settings\<使用者名稱> 的目錄下您能找到一個 Application Data 資料夾；由於該資料夾預設是隱藏的，故請於檔案總管設定顯示隱藏檔。如果您的使用者名稱為「Joe」，且您的 Documents and Settings 資料夾位於 C:\ 之中，則您的個人設定檔便在 C:\Documents and Settings\Joe\Application Data\Mozilla\Firefox 裡，將該 Firefox 資料夾刪除即可。

MacOS X

開啟您 Home folder 中的 Library 資料夾，並將其中的 Firefox 移除。 (~/Library/Firefox/).

其他平台

刪除 ~/.firefox。

您該知道的更動

個人設定檔資料夾

現在 Firefox 0.9 將您的個人設定檔存放在下列位置之一:

WindowsDocuments and Settings\<使用者名稱>\Application Data\Mozilla\Firefox
MacOS X~/Library/Application Support/Firefox
Linux~/.mozilla/firefox/
如果您在升級至 Firefox 0.9 前將 Firefox 0.8 做為預設瀏覽器，那麼您的設定檔將被複製至新的位置，您可自行決定是否將舊的「Phoenix」刪除。

給逐日建置版 (Nightly Build) 的使用者

在 0.8 到 0.9 版的過程中，個人設定檔資料夾曾是 Application Data\Firefox (Windows)、~/Library/Firefox (MacOS X) 及 ~/.firefox (Linux)，而今已經更動至前面所述的位置。在您首次啟動 Firefox 0.9 時，您個人設定檔資料夾中的資訊將複製到新的位置，但不會將舊的刪除。由於新設定檔資料夾的 profiles.ini 中，所有路徑仍然指向舊資料夾，故您必須手動將舊設定資料夾刪除，開啟新的 profiles.ini 並手動更新路徑，使其指向新的設定資料夾。Firefox 0.8 的使用者不必擔心這個問題，這個訊息只是對那些曾冒風險使用尖端技術的測試員說的。

舊的擴充套件及佈景主題

在 0.8 版時部分使用者曾因「No XBL Binding for Browser」錯誤訊息而無法啟動 Firefox，並隨即發現佈景主題失效。為了避免這種問題，在您首次使用 Firefox 0.9 時，所有的擴充套件將自動停用，且佈景主題將重設為預設佈景。您必須另行下載與 Firefox 0.9 新的 API 相容的新版擴充套件。在 Firefox 0.9 版之後相關的更新方式將容易許多。

已知問題

以下列出 Firefox 0.9 的已知問題，請閱讀後再回報新的程式錯誤，並請定期檢查本欄的更新內容﹝如在此版本發現新問題，我們會隨時在[英文網頁]更新此欄﹞。

  • 如果您將 Firefox 0.9 安裝於權限有所管制的區域，那麼就得先由擁有權限的使用者啟動一次 Firefox，以便建立應有的初始啟動檔案。若由權限不足的使用者先行啟動 Firefox，則可能會因為無法建立初始啟動檔案而不斷重新啟動。這個問題將於日後修正。
  • 如果您的 MacOS X 上沒有顯示出 Firefox 的瀏覽視窗，請先用 Cmd+Q 結束 Firefox，然後開啟 ~/Library/Application Support/Firefox/Profiles/default.abc/ 並且將當中的 localstore.rdf 移除，再次重新執行 Firefox。您之前的選項及視窗位置等設定將因而消失。
  • 我們作了一些調整以修正安全性的疏失，避免網站將連結與表單送至另一個網站所製的頁框網頁中；然而這樣的更動卻可能讓某些網站產生錯誤。如果您瀏覽的網站試圖合理使用這項功能，而您亦允許其使用，則請以 about:config 將「docshell.frameloadcheck.disabled」設定為「true」以便略過相關的安全性檢查。如果您碰上這樣的問題，請將造成問題的網址 (URL) 回報至 Bugzilla。
  • Firefox 使用大量 Mozilla Application Suite 的程式碼，許多您可能遇到的問題事實上是 Mozilla 核心程式的問題。如果您發現頁面顯示或連線的問題，它極可能是 Mozilla 的問題，也應被回報到 Bugzilla 的 Browser 產品類而非 Firefox。
  • Windows 上書籤工具列上的選單與資料夾有時不會正確地縮回。
  • 安裝佈景主題時的「使用用這個佈景主題」選項無法正常運作。請使用「工具＞佈景主題」，選擇您所要用的佈景，然後按下「使用佈景主題」以便切換。此外，有時佈景主題在安裝後需要重新開啟程式才能完全套用。
  • 如果您在 0.8 至 0.9 版之間曾使用逐日建置版 (包括 Release Candidate 備選版)，且在啟動 Firefox 時發生不斷重啟的問題，建議您刪除個人設定檔資料夾中的 extensions 資料夾。
  • 部份用戶遇到下載視窗的問題 - 如果下載視窗畫面顯示緩慢，試試[安裝修正檔]
  • 擴充套件及佈景主題的更新系統尚未測試完畢，您可能會碰上些問題，而這些問題預定將於 1.0 beta 版解決。目前update.mozilla.org 尚未部署相關更新服務，也就是說目前更新的功能是無效的。您可能會在無可用更新的情形下，在狀態列看到更新通知。
欲取得其他問題的訊息、常見問題集、使用技巧以及一般 Firefox 協助，請到 David Tenser 非常有用的 [Firefox Help 網站]、[MozillaZine] 的 [Firefox 討論區 (英文)] 或林弘德所維護的 [firefox 線上討論區 (中文)]。

疑難排解

從舊版 Firefox 升級並不需建立一個新的 profile 設定檔，只要您在升級前先將舊版中所有的擴充套件關閉即可。您可以開啟舊版 Firefox，打開「工具＞選項」，然後選擇「擴充套件」，點選列表中每一個擴充套件，並且按下「關閉擴充套件」以將其關閉。按下「確定」關閉選項視窗，便可安全地安裝新版 Firefox 了。

Firefox 安裝完畢，請檢查您使用的擴充套件是否更新以支援新版 Firefox，並下載安裝。未來這個動作將會自動處理。

如果您在舊版 Firebird 或 Phoenix 啟用某些擴充套件並發生了問題，請啟動舊版瀏覽器並將擴充套件關閉，再重新執行新版。

如果您遇到的怪問題與書籤、下載、視窗位置、工具列、瀏覽記錄或其他設定項目有關，建議您在回報問題前先行建立新的使用者設定檔，並查看能否重現該問題。建立新設定檔的方式如下：執行 Firefox 時加上 -P 的命令列參數，選擇「管理使用者設定檔」按鈕，然後選擇「建立使用者設定檔」。將各種設定(書籤、儲存的密碼等)從舊的設定檔中分次轉移到新的設定檔，每移轉一項便檢查是否發生問題。如果您發現某特定設定檔資料會造成問題，請回報錯誤並且附上該檔案。

常見問題集

1. 我該怎麼幫忙呢？

我們需要大量的曝光。讓推廣 Mozilla Firefox 成為您的任務，使您的親朋好友上司下屬都認識它：如果您是個學生，就讓它廣佈校園之中；寫篇文章投稿至 [Slashdot] 等新聞站台上，發布 Firefox 新版的新聞；也可以在您的 Blog 上透漏一點相關訊息；總之，儘可能的在網際網路中散佈，就像 Kazaa 的威力一樣，無所不在！
2. 我是因為某某原因才使用其他瀏覽器的。
如果您不願使用 Firefox，請告訴開發團隊您的理由。我們會持續注意 [Firefox 討論區]中的使用者意見。
3. 你們為什麼都沒有回信？
請使用[討論區]，Firefox 團隊會定期閱讀上面的文章。我們常常都收到大量的信件，而您的可能不小心漏失了。
4. 我該去哪裡取得新的佈景主題與擴充功能？
[擴充功能]與[佈景主題]都可以在 [Mozilla Update]下載。
5. 哪些人正致力於 Firefox 的開發工作？
現在包含 Ben Goodger（為 Mozilla 基金會工作）、Brian Ryner（為 IBM 工作）、Pierre Chanial、Blake Ross、Dave Hyatt、Benjamin Smedberg、Darin Fisher 以及 Mozilla 程式貢獻者們的龐大社群。另外 Asa Dotzler、Leaf Nunes 和一組品質管制團隊等人負責分類程式錯誤、新版釋出及其他雜務。詳細內容請參考「說明＞關於 Firefox」
6. Firefox 的原始碼在何處？
都在 cvs.mozilla.org 。 Mozilla trunk + mozilla/browser + mozilla/toolkit。
7. Firefox 是否只是可調整使用者介面的 Mozilla？
Firefox 在本質上與 Mozilla 有極大差異，除了一些專有的特色之外也在程式上有難以計量的最佳化。自此專案開始後，瀏覽器及 toolkit 的 CVS 目錄中已更動或新增了 120,000 行程式碼。
8. 原來的名字「Phoenix」、「Firebird」又怎麼了？
由於商標的問題我們必需更名，新的名稱叫做「Firefox」。欲尋更多資訊，請見[更名常見問題]。
9. 電子郵件程式呢？
Firefox 能與您預設的任何郵件程式協同運作，而我們推薦您試試我們下一代的郵件軟體：與 Firefox 合作無間的 [Mozilla Thunderbird]。
