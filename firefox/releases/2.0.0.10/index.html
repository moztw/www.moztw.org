Firefox 2.0.0.10 新鮮事

  • 發行日期： 2007 年 11 月 26 日

  • 穩定性更新：修補了數項 安全問題。

  • 前一版：欲知 2.0.0.9 版更新項目，請參考其版本資訊。

下載與安裝

系統需求

在安裝 Firefox 前請先確定您的電腦合乎 系統需求。

下載 Firefox 2

Mozilla.org 提供了 Windows、Linux 及 Mac OS X 平台上多種語言版的 Firefox 2，您可由此下載最新的 Firefox 2。若想自行編譯其他不支援的平台或語言，請見本文件最末節。

安裝 Firefox 2

請注意：安裝 Firefox 2 將覆蓋您現存的 Firefox 程式，這並不會造成書籤或瀏覽記錄遺失，但可能會有些附加元件要更新才能繼續使用。

移除 Firefox 2

在 Windows 作業系統中，您可以自「開始」選單中的「控制台」移除 Firefox 2。OS X 的使用者請移除「Firefox」 程式、Linux 使用者則請移除「firefox」 資料夾。

移除 Firefox 2 並不會將書籤、瀏覽記錄、擴充套件和其他附加元件一併刪除，這些資料會留在您的「個人設定資料夾」中，此資料夾存放位置則視作業系統而定：

Windows Vista Users\<使用者名稱>\AppData\Roaming\Mozilla\Firefox
Windows 2000, XP Documents and Settings\<使用者名稱>\Application Data\Mozilla\Firefox
Windows NT WINNT\Profiles\<使用者名稱>\Application Data\Mozilla\Firefox
Windows 98, ME Windows\Application Data\Mozilla\Firefox
Mac OS X ~/Library/Application Support/Firefox
Linux 及 Unix 系統 ~/.mozilla/firefox

移除 Firefox 2 後所安裝的任一版本 Firefox 將繼續使用個人設定資料夾中的資料。

擴充套件及佈景主題

Firefox 1.5 安裝的擴充套件可能已不相容於 Firefox 2 而應加以升級，如發生此類問題，請回報附加元件原作者。一旦安裝 Firefox 2，所有擴充套件及佈景主題會先停用，待 Firefox 2 偵測出此套件無相容問題或能升級才會啟用。

已知問題

以下列出部分 Firefox 2 已知問題，請閱讀後再回報新的程式錯誤。

所有系統
  • 某些軟體防火牆執行時會悄悄封鎖 Firefox 的對外連線，此類問題通常會在 Firefox 升級或自舊版覆蓋安裝時發生。您可參考防火牆設定步驟確保 Firefox 於網路上通行無阻。
  • 程式中無法內建各地區的不同字典，所以有些拼字檢查用的字典必須要手動下載安裝。Mozilla 附加元件中心常會放上新字典，您可以在文字輸入區中按下滑鼠右鍵，選擇「新增字典」來下載新字典。
  • 在拼字檢查開啟的情況下，若文件列印範圍中包含文字輸入區且輸入區中有錯字，則後續的文件內容都無法印出。列印前您可先在該輸入區上按下滑鼠右鍵，取消「拼字檢查」來暫時避開這個問題。
  • 網頁提供的快速鍵 (Access key) 現在在 Windows 上請以 Alt+Shift+Key 來使用，在 Mac OS X 為 Ctrl+key，Unix 則是 Ctrl+Shift+key。
  • Firefox 2 的瀏覽狀態復原功能在啟動時一樣會保存 cookies 狀態以便在 GMail 那類需要登入的網站保持登入，若您有安全上的顧慮，可以考慮將 browser.sessionstore.resume_from_crash 之值改為 false。
  • 「自動調整圖片大小」的設定項目在 Firefox 2 中刪掉了，如果您想調整這個設定，可以修改 browser.enable_automatic_image_resizing 之值。
  • 若您未擁有 Firefox 安裝處的檔案寫入權限，軟體更新程序便無法進行，這是因為軟體更新必須於安裝處替換或建立檔案。
  • 有些金融機關會使用 563 埠行安全登入之用，這麼一來會出現錯誤訊息。如果您遇到這個問題，請將 563 這個數字加到 network.security.ports.banned.override 的逗點分隔清單中，以便更改內定的通訊埠阻擋設定。
Windows 及 Windows Vista
  • 隨 Java SE 6.0u1 (J2SE6.0.01) 安裝的 Java Console 擴充套件不相容於 Firefox（參考 Bugzilla 上的回報）。Java 本身運行無誤，但「工具」選單中不會出現「Java Console」選項。此問題已於 Java SE 6.0u2 (J2SE6.0.02) 修復，可於 Java 網站上下載更新。
  • 在某些程式中（例如某些即時通訊軟體）按下連結可能無法用 Firefox 開啟網頁，就算你已經設為預設瀏覽器也一樣。要解決這個問題請到「開始＞預設程式＞設定此台電腦的預設程式」，展開「自訂」欄位後選擇你想要設定為預設使用的應用程式（就是 Firefox）後點「套用」即可。
  • Windows Vista 並不提供 Windows Media Player (WMP) plugin，若要觀看 Windows Media 內容，可安裝 WMP Plugin v6.4 for Netscape Navigator。
  • 尚未能完全遵循 Vista 家長控制的設定，下載的檔案並不遵從 Vista 的家長控制設定，這點往後會處理。
  • 自 Internet Explorer 7 轉移至 Firefox 時，cookies 與既存表單內容無法匯入。
  • 以 install-global-extension 及 install-global-theme 命令參數安裝附加元件時，只能安裝已經下載到本地主機 (或已連線之網路磁碟機) 的元件，無法再安裝分享資料夾中的元件。
Mac OS X
  • 在 OS X 10.5（Leopard）上，有些內含 binary 元件的 plugin 或套件可能有問題，此外「偏好設定＞進階」下的頁籤顯示上有問題。
  • 一旦開啟的分頁超過二十個，快捷選單的「關閉其他分頁」功能就可能失效。
  • 有些使用者發現在 Intel 核心的麥金塔電腦上瀏覽 Macromedia Flash 內容會有問題，此時您可以自/Library/Internet Plug-Ins 當中將 PowerPC 版的「Flash Player」刪除。
  • 安裝新 plug-in 後，在 about:plugins 有可能會依然顯示舊版的資訊，此時請離開 Firefox，將個人設定資料夾中的「pluginreg.dat」刪除，然後重新啟動 Firefox。
  • Java 無法於 Intel 核心的 Rosetta 下執行。
  • 在 Intel 核心的麥金塔電腦上執行時，不論原生執行或是透過 Rosetta 執行，都沒有 Talkback（錯誤回報）功能。當錯誤發生時，Apple Crash report 程式會自動執行。
Linux 及 Unix
  • 若 Firefox 的安裝路徑中含有空白字元，可能便無法設定為預設瀏覽器、並將於啟動時不斷詢問是否設定。目前的解決方法是安裝到無空白字元的路徑下。
  • 在 Fedora Core 3 無法與 GNOME 正常整合。 Fedora Core 3 的使用者必須先下載 linc-1.0.3-3.1.i386.rpm 後安裝，安裝完畢後在 Firefox 的安裝目錄下執行此命令 (需要寫入權限)：

    touch .autoreg

    下回使用 Firefox 時，便應該能正確整合至 GNOME 環境。

疑難排解

  • 設計不良或不相容的套件將導致瀏覽器發生問題，發生的問題包含當機或網頁瀏覽速度變慢等等。如果您碰上瀏覽器無法開啟、視窗顯示不正常等等狀況，便可能是擴充套件或佈景主題的問題。這時，請將瀏覽器重新啟動在安全模式：
    • 在 Windows 下用「Safe Mode」捷徑或在命令列中輸入 firefox.exe -safe-mode 指令。
    • 在 Linux 下請執行./firefox -safe-mode 指令。
    • 在 MacOS X 下請輸入

      cd /Applications/Firefox.app/Contents/MacOS/
      ./firefox-bin -safe-mode

    當啟動安全模式時，所有的附加元件將停用並使用預設的佈景主題。在安全模式下移除有問題的擴充套件/佈景主題，然後再正常的啟動瀏覽器。
  • 如果您想在移除某擴充套件後，以 -install-global-extension 指令將同一擴充套件裝到每個使用者設定檔中，則必須在重新安裝前先重開瀏覽器一次，以便徹底移除擴充套件。如果您沒這麼做，則擴充套件清單可能會有衝突項目而不能為所有設定檔安裝。
  • 如果您遇到的怪問題與書籤、下載、視窗位置、工具列、瀏覽記錄或其他設定項目有關，建議您在回報問題前先行建立新的使用者設定檔，並查看能否重現該問題。建立新設定檔的方式如下：執行 Firefox 時加上 -P 的命令列參數，選擇「管理使用者設定檔」按鈕，然後選擇「建立使用者設定檔」。將各種設定(書籤、儲存的密碼等)從舊的設定檔中分次轉移到新的設定檔，每移轉一項便檢查是否發生問題。如果您發現某特定設定檔資料會造成問題，請回報錯誤並且附上該檔案。

常見問題集

  1. 我該怎麼幫忙呢？

    我們需要開發員和測試社群回饋建議，幫我們將 Firefox 變得更盡善盡美。請先閱讀錯誤回報注意事項，再到 Bugzilla 回報。您也可以使用回饋表單表達意見。

  2. 你們為什麼都沒有回信？

    請使用討論區， Firefox 團隊會定期閱讀上面的文章。我們常常都收到大量的信件，而您的可能不小心漏失了。

  3. 我該去哪裡取得新的佈景主題與擴充功能 (附加元件)？

    擴充套件及佈景主題可自 Mozilla 附加元件中心下載。

  4. Firefox 2 是哪些人做出來的？

    很多人。「說明＞關於 Mozilla Firefox」中有份名單，列出部分曾對 Firefox 2 有所貢獻的人。

  5. Firefox 2 的原始碼在何處？

    Firefox 2 原始碼已開放下載，而目前開發中的最新程式碼則需以 CVS 取得。Firefox 相關程式為「mozilla/browser」、「mozilla/toolkit」及「mozilla/chrome」，請參考建製步驟。

  6. 電子郵件程式呢？

    Firefox 2 能與您預設的任何郵件程式協同運作， 而我們推薦您試試我們下一代的郵件軟體：與 Firefox 合作無間的 Mozilla Thunderbird。
